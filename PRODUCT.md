# Product

## Register

product

## Users

星途新媒体运营团队、业务负责人和领导层。运营团队需要在飞书固定入口里快速核对账号曝光、发布表现、演员参与和单条视频榜单；业务负责人和领导层需要在浅色、正式、稳定的管理驾驶舱里扫清核心 KPI 和异常视频表现。

## Product Purpose

将已验收的短视频 Excel 口径固定为一个独立网页 BI，支持手动上传或替换 Excel 后刷新数据。首版成功标准是账号维度和演员维度两张汇总表稳定可读，总览 KPI、Top1/Bot1 单条视频榜单、账号表现和演员表现能够被领导和运营直接用于复盘，不展示 sheet、字段缺失、数据质量等技术性解释。

## Brand Personality

克制、清晰、可信。

界面语气应像企业内部经营看板：少装饰、强层级、指标优先、口径稳定。视觉上保持浅色企业 BI，不做深色大屏，也不做营销页式夸张表达。

## Anti-references

- 不做深色大屏、炫光大屏或舞台式数据可视化。
- 不展示 sheet、字段来源、缺失字段、fallback 等技术性解释。
- 不使用复杂 BI 平台式配置面板、权限模型或飞书 API 同步叙事。
- 不使用一屏堆满同质卡片、过度阴影、过圆容器、渐变文字或装饰性动效。
- 不把数据缺失智能补齐；视频号没有 5S 完播率时保持未提供，多人演员贡献曝光量不平摊。

## Design Principles

1. 管理驾驶舱优先：首屏先回答总量、效率、演员参与和榜单变化，再进入明细。
2. 口径可被复盘：所有指标表达贴合已确认口径，数值超过五位数按万展示。
3. 运营可连续使用：导航、筛选、表格和上传入口都要稳定、低学习成本、适合飞书 iframe。
4. 正式但不沉闷：使用浅色企业风格、清晰分区和克制强调色，让领导扫读时有重点。
5. 最少技术噪音：页面只呈现业务结论和必要操作，不把技术解释放进报表主体。

## Accessibility & Inclusion

按 WCAG AA 作为首版目标：正文与背景对比度不低于 4.5:1，交互控件必须有可见焦点状态，图表不能只靠颜色表达，移动端和飞书窄 iframe 下不出现文字溢出或横向页面级滚动。动效只用于状态反馈，并尊重 reduced motion。
